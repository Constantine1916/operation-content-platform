# AI树洞未登录落地页设计稿

- Date: 2026-04-19
- Product: AI树洞
- Domain: aicave.cn
- Page: 未登录落地页
- Scope: Stitch desktop landing page first pass

## 1. Context

AI树洞是一个 AI 资源分享平台，覆盖资讯、文章、图片、视频与课程。当前首页结构完整，但视觉重心偏信息陈列，第一屏更像功能介绍，不足以在访问者进入页面的前几秒建立强烈品牌印象，也没有把“AI树洞为什么存在”讲成一个足够有记忆点的概念。

这次 redesign 的目标不是把首页做成普通 SaaS 官网，而是把 AI树洞塑造成一个通往 AI 资源世界的入口。页面需要先建立炫酷、沉浸、未来感的第一印象，再用克制的文案解释平台理念和资源范围。

## 2. Design Goals

1. 让未登录用户在第一屏感受到强烈、炫酷、可信的品牌气质。
2. 在较少文案内解释 AI树洞的核心理念。
3. 让用户明确理解网站提供多种 AI 资源，而不是单一工具或单一内容类型。
4. 同时兼顾资源浏览者与创作者，但叙事顺序先打动寻找资源的人。
5. 产出一版可直接在 Stitch 中生成的高保真 landing page 方案。

## 3. Non-Goals

1. 不在首屏展示真实 dashboard 或密集内容卡片。
2. 不走通用 SaaS 官网路线。
3. 不做高噪音赛博朋克，不做紫色霓虹主导视觉。
4. 不在这轮设计中优先解决登录后产品信息架构。
5. 不在这轮设计中优先处理完整移动端细节，先稳定 desktop hero 与页面叙事。

## 4. Audience

### Primary Audience

寻找 AI 资源的人。他们想更快发现值得看的 AI 资讯、文章、图片、视频和课程，希望少花时间在分散的信息流之间来回切换。

### Secondary Audience

AI 创作者。他们希望上传、展示、分发自己的内容和作品，并触达真正对 AI 内容感兴趣的人群。

## 5. Core Positioning

AI树洞不是“AI 内容列表页”，而是一处“进入 AI 资源宇宙的洞口”。

品牌隐喻：

- “树洞”意味着沉淀、连接、发现与进入。
- “洞口”意味着入口、探索与空间深度。
- “AI 资源入口”意味着把分散的内容信号汇聚到一个持续更新、可探索的地方。

## 6. Visual Thesis

用“未来洞穴 / 数字宇宙”作为整体母题，把 AI树洞做成一个安静、深邃、迷人的入口空间。

关键词：

- future cave
- digital universe
- cinematic depth
- obsidian darkness
- fog and volumetric light
- cyan-blue fissure
- sparse signal particles
- premium restraint

页面应更接近沉浸式创意工具官网，而不是传统产品介绍站。

## 7. Content Plan

页面按照四段叙事展开：

1. Hero
   负责建立品牌世界观与第一印象。
2. Support
   用“资源星图”解释 AI树洞能提供哪些资源。
3. Detail
   用品牌理念区和双边价值区解释 AI树洞为什么存在、对谁有价值。
4. Final CTA
   引导用户进入站点，开始探索或注册。

## 8. Interaction Thesis

页面只保留少量但有存在感的动效：

1. Hero 主视觉轻微呼吸脉冲，像洞口在发光。
2. 背景雾层、光点和深度平面缓慢漂移，形成空间纵深。
3. 资源星图在滚动时逐个唤醒，让用户感觉自己正在进入更深层的资源空间。

动效必须慢、克制、平滑，不能做成 flashy tech demo。

## 9. Information Architecture

整页控制为五个主分区：

1. Hero: 进入洞口
2. Resource Constellation: 你能在这里找到什么
3. Brand Philosophy: 为什么叫 AI树洞
4. Dual Audience Value: 浏览者与创作者
5. Final CTA: 现在，进入 AI树洞

## 10. Page Sections

### 10.1 Top Navigation

导航应为极简 overlay 形式，避免厚重顶栏占用首屏空间。

Items:

- Logo text: AI树洞
- Nav link: 资源
- Nav link: 创作者
- Nav link: 登录
- Primary CTA: 进入树洞

要求：

- 导航轻、细、透明，不抢 hero。
- 品牌必须在第一屏足够明确。

### 10.2 Hero

Hero 是整页核心，必须 edge-to-edge，不使用居中盒状主容器。

构图要求：

- 全屏沉浸式首屏。
- 中心或略偏中的区域出现一个发光的洞口 / 裂隙 / 门廊式空间。
- 背景为深黑、石墨黑、深蓝灰的层次空间。
- 局部有雾、体积光、远处信号点，形成深度。
- 文案放在一个视觉上相对安静、可读的区域。

Hero 文案固定如下：

- Brand: AI树洞
- Headline: 进入 AI 资源的洞口
- Supporting text: 把分散的资讯、文章、图片、视频与课程，汇聚成一个可探索、可发现、可持续更新的 AI 资源入口。
- Primary CTA: 进入树洞
- Secondary CTA: 浏览资源
- Resource labels: AI资讯 / 深度文章 / 灵感图片 / AI视频 / 实战课程

Hero 责任：

1. 建立第一印象。
2. 解释 AI树洞的理念。
3. 用最少文案说明资源范围。
4. 提供清晰行动入口。

### 10.3 Resource Constellation

这一屏不做功能卡片网格，而是做“资源星图”。

表现方式：

- 抽象星图、轨道或信号节点式布局。
- 五个资源类型作为光点节点被组织在一个有秩序的资源图谱中。
- 视觉上看起来像发现不同资源入口，而不是读一个列表。

资源类型与短说明：

- AI资讯: 追踪 AI 世界正在发生什么
- 深度文章: 理解趋势、方法与观点
- 灵感图片: 收集生成灵感与美学样本
- AI视频: 看见 AI 作品与表达方式
- 实战课程: 把工具真正用起来

责任：

1. 讲清平台提供多种 AI 资源。
2. 延续世界观，而不是跳出到普通模块化官网。

### 10.4 Brand Philosophy

这是较安静的一屏，用来解释“为什么叫 AI树洞”。

建议文案：

在信息过载的 AI 时代，人们需要一个能沉淀、发现、连接资源的入口。AI树洞希望成为这样一个地方。

视觉要求：

- 更安静、更开阔的深层洞室感。
- 减少强光和装饰，突出文字与空间感。
- 让用户在阅读时从炫酷状态过渡到理解状态。

责任：

1. 给品牌命名和概念一个明确解释。
2. 把“炫酷”转化为“可信的品牌叙事”。

### 10.5 Dual Audience Value

这一屏同时承接浏览者和创作者，但语义上先强调浏览者。

左侧：

- 寻找资源的人
- 更快发现值得看的 AI 内容与作品

右侧：

- 创作者
- 上传、展示、连接对你内容真正感兴趣的人

视觉要求：

- 不做 pricing card 式双卡片。
- 可使用对照式排版、纵向分栏、光线分区或路径分流设计。
- 浏览者与创作者应被理解为同一生态中的两个角色，而不是两个割裂套餐。

### 10.6 Final CTA

最后一屏应回归洞口意象，但更简洁，作为情绪收束。

文案：

- Headline: 现在，进入 AI树洞
- Supporting text: 从发现资源开始，也从分享资源开始。
- Primary CTA: 进入树洞
- Secondary CTA: 登录或注册

责任：

1. 完成品牌记忆收束。
2. 提供明确行动出口。

## 11. Visual System

### Color

- Background base: `#07090d` to `#0f1117`
- Primary glow: cool cyan / electric blue
- Highlight: soft silver-white
- Support fog gray: `#8b95a7`

约束：

- 不引入紫色主轴。
- 不使用高饱和多彩渐变做壁纸。
- 强调冷、深、静的空间感。

### Material

材质感觉应来自：

- 黑曜石岩壁
- 冷雾
- 发光裂隙
- 远处信号点

禁用：

- 赛博都市街景
- 机械面板
- 浮动卡片堆
- 过强金属科幻 UI

### Typography

- 标题：有文学感、雕塑感或编辑感的中文 display / serif 风格
- 正文与导航：干净、克制的 sans-serif
- 全页最多两种字体气质

### Motion

- Hero glow breathing
- layered depth drift
- scroll-based constellation reveal

交互反馈应为极轻微 hover、微上浮、微发光，不做强缩放或过度弹性。

## 12. Hard Rules

1. No generic SaaS blocks.
2. No hero cards.
3. No dashboard mockups.
4. No purple cyberpunk aesthetic.
5. No floating UI panels.
6. No visual clutter.
7. Chinese text readability must remain strong.
8. The first screen must feel like a poster, not a document.

## 14. Delivery Notes

1. Generate desktop first in Stitch.
2. Review whether the hero worldbuilding is strong enough before generating variants.
3. If the first result drifts into generic SaaS patterns, tighten the prompt around “poster-like hero”, “no cards”, and “abstract cave portal composition”.
4. If needed, generate a second variant with stronger negative space and fewer UI elements rather than adding more sections.
